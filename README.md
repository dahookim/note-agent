# 🧠 OSBA - Obsidian Second Brain Agent

> AI 기반 세컨드 브레인 자동화 플러그인

당신의 Obsidian 볼트를 더 스마트하게 만들어주는 AI 어시스턴트입니다. 노트 간의 숨겨진 연결을 발견하고, 지식 갭을 분석하며, 컨텍스트 기반의 새로운 콘텐츠를 생성합니다.

## ✨ 주요 기능

### 🔗 자동 연결 발견
- **벡터 임베딩 기반**: 의미적으로 유사한 노트들을 자동으로 찾아냅니다
- **관계 유형 분류**: extends, supports, contradicts, examples, related 등 관계 유형을 식별합니다
- **신뢰도 점수**: 각 연결의 확신도를 퍼센트로 표시합니다

### 🔍 지식 갭 분석
- 노트에서 부족한 부분이나 더 탐구할 만한 주제를 발견합니다
- 우선순위별로 지식 갭을 분류합니다 (높음/중간/낮음)
- 추가 학습을 위한 리소스를 제안합니다

### ✨ Quick Draft (빠른 초안)
- 관련 노트들의 컨텍스트를 활용하여 새로운 콘텐츠를 생성합니다
- RAG (Retrieval-Augmented Generation) 기반으로 정확한 정보를 제공합니다
- 새 노트로 저장하거나 현재 노트에 삽입할 수 있습니다

### 📚 유사 노트 찾기
- 현재 노트와 의미적으로 유사한 노트들을 검색합니다
- 유사도 퍼센트와 함께 결과를 표시합니다
- 놓친 연결을 발견하는 데 유용합니다

### 📊 배치 인덱싱
- 볼트 전체를 한 번에 인덱싱합니다
- 진행 상황을 실시간으로 확인할 수 있습니다
- 캐시를 활용하여 이미 처리된 노트는 건너뜁니다

## 📦 설치 방법

### BRAT을 통한 설치 (권장)

BRAT (Beta Reviewers Auto-update Tester)를 사용하면 GitHub에서 직접 베타 플러그인을 설치할 수 있습니다.

#### 1단계: BRAT 플러그인 설치
1. Obsidian 열기
2. `설정` → `커뮤니티 플러그인` → `찾아보기`
3. "BRAT" 검색
4. 설치 후 활성화

#### 2단계: OSBA 플러그인 추가
1. `설정` → `BRAT` → `Add Beta plugin`
2. 다음 저장소 주소 입력:
   ```
   reallygood83/obsidian-ai-agent
   ```
3. `Add Plugin` 클릭

#### 3단계: 플러그인 활성화
1. `설정` → `커뮤니티 플러그인`
2. "Second Brain Agent" 찾기
3. 토글을 켜서 활성화

## ⚙️ 초기 설정

플러그인을 사용하기 전에 AI 제공자를 설정해야 합니다.

### API 제공자 설정

`설정` → `Second Brain Agent` → `API 설정`

#### OpenAI (권장)
```
제공자: OpenAI
API 키: sk-xxxxxxxxxxxx
채팅 모델: gpt-4o-mini (비용 효율적) 또는 gpt-4o (고품질)
임베딩 모델: text-embedding-3-small (권장)
```

#### Anthropic (Claude)
```
제공자: Anthropic
API 키: sk-ant-xxxxxxxxxxxx
채팅 모델: claude-3-5-sonnet-20241022
```

#### Ollama (로컬, 무료)
```
제공자: Ollama
Base URL: http://localhost:11434
채팅 모델: llama3.2
임베딩 모델: nomic-embed-text
```

### 임베딩 설정

| 설정 | 권장값 | 설명 |
|------|--------|------|
| 임베딩 차원 | 1536 | OpenAI text-embedding-3-small 기준 |
| 배치 크기 | 10 | 한 번에 처리할 노트 수 |
| 자동 임베딩 | ON | 노트 저장 시 자동 임베딩 |
| 디바운스 | 5000ms | 저장 후 임베딩 대기 시간 |

### 제외 설정

인덱싱에서 제외할 폴더나 파일 패턴을 설정합니다:

```
제외 폴더: Templates, Archive, .obsidian
제외 패턴: *.excalidraw.md, daily-*.md
```

## 📖 사용 가이드

### 🚀 시작하기

#### 1. 볼트 인덱싱

처음 사용할 때는 볼트 전체를 인덱싱해야 합니다:

1. 명령어 팔레트 열기 (`Cmd/Ctrl + P`)
2. "OSBA: Batch Index Vault" 검색 및 실행
3. 진행 상황 모달에서 완료될 때까지 대기

> 💡 **팁**: 인덱싱은 노트 수에 따라 시간이 걸릴 수 있습니다. 백그라운드에서 실행되므로 다른 작업을 계속할 수 있습니다.

#### 2. 노트 분석하기

특정 노트의 연결과 지식 갭을 분석합니다:

1. 분석할 노트 열기
2. 명령어 팔레트에서 "OSBA: Analyze Current Note" 실행
3. 분석 결과 모달에서 내용 확인

**분석 결과에 포함되는 내용:**
- 🔗 발견된 연결 (관련 노트와의 관계)
- 🔍 지식 갭 (더 탐구할 주제)
- 💡 인사이트 (AI가 제안하는 개선점)

#### 3. 유사 노트 찾기

현재 노트와 비슷한 노트들을 검색합니다:

1. 기준이 될 노트 열기
2. "OSBA: Find Similar Notes" 실행
3. 유사도 순으로 정렬된 결과 확인

#### 4. Quick Draft 사용하기

관련 노트들의 컨텍스트를 활용해 새 콘텐츠를 생성합니다:

1. "OSBA: Quick Draft" 실행
2. 작성하고 싶은 내용 설명 입력
   - 예: "AI 기반 노트 관리 시스템의 장단점을 정리해줘"
3. "생성하기" 클릭
4. 결과 확인 후 선택:
   - **새 노트로 저장**: 독립적인 새 노트 생성
   - **클립보드에 복사**: 다른 곳에 붙여넣기용
   - **현재 노트에 삽입**: 열려있는 노트 끝에 추가

## 💰 비용 관리

OSBA는 AI API를 사용하므로 비용이 발생합니다. 효율적인 비용 관리를 위한 가이드입니다.

### 비용 대시보드

`OSBA: Open Cost Dashboard` 명령어로 열 수 있습니다.

**표시 정보:**
- 오늘/이번 주/이번 달 총 비용
- 제공자별 사용량
- 모델별 비용 분석
- 작업 유형별 비용

### 비용 절약 팁

#### 1. 적절한 모델 선택
| 용도 | 권장 모델 | 비용 |
|------|-----------|------|
| 임베딩 | text-embedding-3-small | $0.02/1M tokens |
| 분석 (일반) | gpt-4o-mini | $0.15/1M input |
| 분석 (고품질) | gpt-4o | $2.50/1M input |

#### 2. 제외 설정 활용
- 템플릿 폴더 제외
- 일일 노트 패턴 제외
- 자주 변경되지 않는 아카이브 제외

#### 3. 자동 임베딩 조절
- 디바운스 시간을 늘려 불필요한 재처리 방지
- 필요한 경우에만 수동 인덱싱 실행

#### 4. Ollama 활용 (무료)
- 로컬에서 실행되므로 API 비용 없음
- 인터넷 연결 없이 사용 가능
- 단, 하드웨어 요구사항 확인 필요

### 예상 비용 (OpenAI 기준)

| 작업 | 예상 비용 |
|------|-----------|
| 노트 1개 임베딩 | ~$0.0001 |
| 노트 1개 분석 | ~$0.001-0.01 |
| Quick Draft 1회 | ~$0.005-0.02 |
| 100개 노트 배치 인덱싱 | ~$0.01-0.02 |

## 🔧 문제 해결

### 일반적인 문제들

#### "API 키가 유효하지 않습니다"
- API 키가 올바르게 입력되었는지 확인
- 키 앞뒤 공백 제거
- 제공자에 맞는 키 형식인지 확인:
  - OpenAI: `sk-` 또는 `sk-proj-`로 시작
  - Anthropic: `sk-ant-`로 시작

#### "임베딩 생성 실패"
- 인터넷 연결 확인
- API 잔액/크레딧 확인
- 노트 내용이 너무 길지 않은지 확인 (토큰 제한)

#### "유사한 노트를 찾을 수 없습니다"
- 먼저 "Batch Index Vault"로 볼트를 인덱싱했는지 확인
- 제외 설정에 해당 노트가 포함되지 않았는지 확인

#### "분석 결과가 나타나지 않습니다"
- 노트가 충분한 내용을 포함하고 있는지 확인
- 제외된 폴더에 있지 않은지 확인
- 콘솔에서 오류 메시지 확인 (`Cmd/Ctrl + Shift + I`)

#### Ollama 연결 실패
1. Ollama가 실행 중인지 확인:
   ```bash
   ollama list
   ```
2. 모델이 설치되어 있는지 확인:
   ```bash
   ollama pull llama3.2
   ollama pull nomic-embed-text
   ```
3. Base URL이 올바른지 확인 (기본: `http://localhost:11434`)

### 성능 최적화

#### 대용량 볼트 (1000+ 노트)
- 배치 크기를 5-10으로 설정
- 불필요한 폴더 제외
- 점진적으로 인덱싱 (폴더별)

#### 느린 분석 속도
- gpt-4o-mini 같은 빠른 모델 사용
- 분석 깊이 설정 조절
- 캐시 활용 (이미 분석된 노트는 빠름)

## ❓ FAQ

### Q: 어떤 AI 제공자를 선택해야 하나요?

**OpenAI (권장)**
- 가장 안정적이고 빠름
- 다양한 모델 선택 가능
- 한국어 지원 우수

**Anthropic (Claude)**
- 긴 컨텍스트 지원
- 분석 품질 우수
- 상대적으로 비용 높음

**Ollama**
- 완전 무료
- 프라이버시 보장 (로컬 실행)
- 설정이 복잡하고 하드웨어 요구

### Q: 노트 내용이 외부로 전송되나요?

네, AI 분석을 위해 노트 내용이 선택한 AI 제공자에게 전송됩니다.
- **OpenAI/Anthropic**: 클라우드 서버로 전송
- **Ollama**: 로컬에서만 처리 (외부 전송 없음)

민감한 정보가 있는 노트는 제외 설정에 추가하세요.

### Q: 임베딩 데이터는 어디에 저장되나요?

볼트 내 `.obsidian/plugins/obsidian-second-brain-agent/` 폴더에 SQLite 데이터베이스로 저장됩니다. 이 데이터는 다른 곳으로 전송되지 않습니다.

### Q: 이미 있는 노트의 연결과 어떻게 다른가요?

Obsidian의 기본 백링크는 명시적인 `[[링크]]`만 인식합니다. OSBA는:
- 링크 없이도 **의미적 유사성**을 찾습니다
- 관계의 **유형**을 분류합니다
- 연결의 **이유**를 설명합니다
- **놓친 연결**을 발견합니다

### Q: 비용 한도를 설정할 수 있나요?

현재 버전에서는 비용 한도 기능이 없습니다. 대신:
- Cost Dashboard에서 사용량 모니터링
- 제외 설정으로 처리량 제한
- API 제공자의 사용량 한도 설정

### Q: 플러그인을 제거하면 데이터는 어떻게 되나요?

- 임베딩 데이터베이스는 플러그인 폴더와 함께 삭제됩니다
- 노트에 추가된 Connected Insights 섹션은 유지됩니다
- frontmatter에 추가된 메타데이터는 유지됩니다

### Q: 모바일에서도 사용할 수 있나요?

아니요, OSBA는 데스크톱 전용 플러그인입니다. SQLite와 네이티브 Node.js 모듈을 사용하기 때문입니다.
